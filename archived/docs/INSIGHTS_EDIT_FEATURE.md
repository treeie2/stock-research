# 📝 编辑功能说明

## 功能说明

在个股详情页（`/stock/{code}`）可以编辑以下字段：
- ⚡ **催化剂 / 事件驱动**（accident）
- 💡 **投资洞察**（insights）

**只读字段**（不可编辑）：
- 概念标签、产品/服务、合作伙伴、产业链位置、关键指标等

## 使用方法

### 1. 开启编辑模式
1. 访问任意股票详情页，如：https://web-production-a1006c.up.railway.app/stock/300308
2. 在导航栏右侧找到 **「🔒 编辑模式：关闭」** 按钮
3. 点击按钮，切换为 **「🔓 编辑模式：开启」**
4. 此时可编辑的字段会显示 **「✏️ 编辑」** 按钮

### 2. 编辑内容
1. 点击要编辑字段的 **「✏️ 编辑」** 按钮
2. 在文本框中修改内容
3. 支持多来源格式（可选）：
```
[1] 第一个来源的内容...

---

[2] 第二个来源的内容...
```

### 3. 保存
1. 点击 **「💾 保存」** 按钮
2. 等待保存成功提示（绿色）
3. 自动切换回显示模式
4. 系统自动重建搜索索引

### 4. 关闭编辑模式
- 编辑完成后，点击导航栏的 **「🔓 编辑模式：开启」** 按钮
- 切换回 **「🔒 编辑模式：关闭」**
- 编辑按钮隐藏，防止误触

### 2. 编辑内容
- 在文本框中输入或修改投资洞察内容
- 支持多来源格式：
```
[1] 第一个来源的内容...

---

[2] 第二个来源的内容...

---

[3] 第三个来源的内容...
```

### 3. 保存
- 点击 **「💾 保存」** 按钮
- 等待保存成功提示（绿色）
- 自动切换回显示模式
- 系统自动重建搜索索引

### 4. 取消编辑
- 点击 **「取消」** 按钮
- 不保存任何更改

## 多来源格式说明

为了保持数据一致性，建议使用以下格式：

```
[1] 来源一的内容

---

[2] 来源二的内容

---

[3] 来源三的内容
```

- `[1]`, `[2]`, `[3]`：来源序号
- `---`：来源分隔线（前后各一个空行）
- 每个来源独立成段

## 技术实现

### 前端
- 切换显示/编辑模式
- 实时保存状态提示
- 自动刷新显示内容

### 后端 API
- **端点**: `PUT /api/stock/{code}/insights`
- **请求体**: `{"insights": "内容"}`
- **响应**: `{"success": true, "message": "已保存"}`

### 数据存储
1. 更新 `data/master/stocks_master.json`
2. 自动重建 `data/sentiment/search_index_full.json.gz`
3. Git 自动提交（待实现）

## 注意事项

1. **权限**：当前无权限控制，任何人可编辑（建议未来添加认证）
2. **版本**：暂不支持版本回退
3. **审核**：编辑后建议手动验证
4. **备份**：重要修改前建议手动备份数据
5. **编辑模式**：默认关闭，需要手动开启防止误触
6. **只读字段**：概念标签、产品、合作伙伴等不可编辑
